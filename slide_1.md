# Slide 1 Specification

## Slide Purpose
This is the opening visual sequence of the talk.

Its job is to make the audience immediately recognize the emotional pattern of building with AI:
- initial excitement
- confidence
- momentum
- first unexpected failure
- attempted recovery
- cascading failure
- total collapse

This slide should not explain the whole thesis.
It should make the audience feel the thesis before it is named.

## Where It Appears in the Talk
This slide comes **after** the opening hand-raise interaction.

The live order is:
1. Speaker asks audience who has used AI to build something that worked.
2. Speaker asks who then tried to keep building on it and had things start breaking.
3. This slide appears and unfolds step by step.

So the audience should already be primed to think: "yes, this happened to me."

## Core Sequence (locked)
Reveal the following sequence one step at a time, from left to right:

Build -> Fix -> Add -> Add -> Fix -> Add -> Break -> Fix -> Break -> Break -> Break

Each item should appear one at a time on click, tap, or key press.
Each step should remain visible after appearing.

## Interaction Model
This slide must support progressive reveal.
Each reveal should feel like one beat in a story.

### Reveal order
1. Build
2. Fix
3. Add
4. Add
5. Fix
6. Add
7. Break
8. Fix
9. Break
10. Break
11. Break

Use either:
- click anywhere to advance
- keyboard right arrow / space to advance

The animation should be subtle and quick. No flashy transitions.
Prefer fade + slight upward motion, or simple opacity reveal.

## Emotional Arc
The emotional journey should feel like this:

1. Build = euphoria
2. Early Fix / Add / Add / Fix / Add = momentum and growing confidence
3. First Break = first crack / concern
4. Fix after Break = false confidence / trying to regain control
5. Final three Breaks = accelerating instability and collapse

The slide should make the audience laugh in recognition, not because it is cartoonish.

## Layout
The layout should be horizontal and highly legible.

### Recommended structure
- Full-screen slide
- Neutral background
- Main sequence centered horizontally, roughly mid-screen
- Strong spacing between words and arrows early on
- Sequence grows left to right

### Important visual behavior
As the final three Breaks appear, the layout should subtly feel less stable:
- slightly less even spacing
- a touch more visual tension
- final break visually strongest

Do not make it messy too early.
The breakdown should escalate.

## Typography
- Very large type
- Clean sans serif
- Bold or semi-bold
- Easy to read from far away
- Single-line sequence if possible

Words should be significantly larger than any meme captions or auxiliary visuals.

## Color System
Use color intentionally to reinforce state.

### Suggested logic
- Build / Fix / Add = neutral light text on dark background, or dark text on light background
- Break = red or failure color
- Final Breaks = increasingly intense red emphasis

Do not over-color the whole slide.
The red should feel meaningful because it is restrained until failure appears.

## Meme Strategy
This slide should include a few memes, but only on specific beats.
The memes are there to externalize emotion, not to dominate the slide.

### Meme placement
Use memes only on these moments:
- Build
- first strong Add confidence beat
- first Break
- Fix after first Break
- final Break

This means around 5 meme moments total.

### Emotional mapping
1. Build
   - emotion: "I am a genius"
   - style: galaxy brain, god-tier confidence, legendary success, absurd self-belief

2. First major Add confidence moment
   - emotion: "I am locked in"
   - style: focused, flow state, typing mode, fully in control

3. First Break
   - emotion: "uh oh"
   - style: confusion, blinking, concern, error realization

4. Fix after first Break
   - emotion: "ok I got this"
   - style: determined recovery, trying to hold it together

5. Final Break
   - emotion: full meltdown
   - style: rage, collapse, enraged disbelief, everything is broken

### Meme integration rules
- Memes should appear near the relevant step, not cover the whole screen
- Early memes should be smaller
- Final meltdown meme can be larger and more dominant
- Memes should feel integrated into the deck aesthetic, not pasted in randomly
- Use tasteful cropping or stylized framing if needed

## Animation Guidance
Sequence reveals should be quick.
Meme reveals can be slightly more dramatic than text reveals, but still restrained.

### Suggested behavior
- text / arrows: quick fade in
- memes: quick pop or scale-in, then settle
- no bounce, spin, or silly animation

The final Break should feel heavier than the earlier reveals.
A slightly slower reveal or larger scale is acceptable.

## Speaker Timing Support
This slide is meant to support a short spoken sequence, not a long explanation.

The speaker will say roughly:
- It starts like this.
- You build something. It works.
- You fix things. Add features.
- Still feels great.
- Then something breaks.
- You fix it.
- Something else breaks.
- Then something else breaks.
- Then something else breaks.
- At this point, you are not building anymore. You are just trying to keep it from falling apart.

The slide pacing should make those beats feel natural.

## Important Final Beat
After the last Break appears, hold the final state on screen.
This is where the audience should have a recognition laugh.

The final state should feel like:
- confidence is gone
- chaos has taken over
- the builder is now trapped in reaction mode

This is the emotional setup for the next slide.

## Transition to Next Slide
The next slide is the Dory analogy slide.
So this slide should end at the precise emotional point where the speaker can say:

"This is what it feels like. It's like building with Dory."

That transition should feel obvious and satisfying.

## Do Not
- do not explain the whole thesis on this slide
- do not include paragraphs
- do not use more memes than specified
- do not make the slide visually noisy too early
- do not let memes overpower readability
- do not turn the slide into a generic meme collage
- do not use chaotic animation
- do not add technical jargon

## Success Criteria
The slide is successful if:
- the audience instantly recognizes the pattern
- the escalation feels emotionally true
- the memes amplify recognition instead of distracting
- the final triple Break lands hard
- the slide sets up the next idea without needing explanation
