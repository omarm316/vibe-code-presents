# Slide 4 Specification

## Slide Purpose
Establish credibility by showing this failure is a **repeatable pattern** across multiple builds and across multiple AI tools.

This slide answers: **Is this just me, or is this real?**

It should feel like proof, not a résumé.

## Core Idea (locked)

I kept hitting this  
Across projects and tools

## On-Slide Text (locked)

I kept hitting this  
Across apps, games, tools, sites

(Keep to two lines. No extra words.)

## Visual Design (locked)

### Primary Visual
A clean, minimal grid of 4 icons (top row):
- phone (apps)
- game controller (games)
- dashboard (internal tools)
- browser (websites)

Below (secondary, smaller, lighter): simple text labels for tools:
- Claude
- Codex
- Cursor
- Antigravity

### Layout
- Full-screen slide
- Text centered near top or center
- Icons centered beneath text with generous spacing
- Tool names in a subtle row below icons (smaller, lower contrast)

### Visual Principles
- high whitespace
- no logos or screenshots
- no clutter
- tools are **supporting evidence**, not the headline

## Typography
- very large font (>= 40pt equivalent)
- bold or semi-bold for main text
- clean sans serif

## Color
- neutral background
- high contrast for main text
- lower contrast for tool names
- no red emphasis on this slide

## Animation

### Entrance
- first line appears first: `I kept hitting this`
- second line builds one word at a time:
  `Across apps,` then `games,` then `tools,` then `sites`
- each matching icon appears with its word
- tool names fade in last (subtle)

### Behavior
- calm, controlled
- no flashy transitions
- each word/icon beat should feel like accumulating evidence, not a reset

## Speaker Script (guideline)

Say approximately:

"I didn’t just run into this once…"

"I hit it across everything I built with AI—apps, games, internal tools, websites…"

(pause)

"And I kept thinking—maybe it’s the tool."

"So I tried different ones… Claude, Codex, Cursor, Antigravity…"

(pause)

"Same problem. Every time."

Then give ONE concrete example (locked):

"We’d add a feature—like reacting to posts—and it would show up on the feed…
…but not on the comment view…
…and totally break sharing."

(pause)

"That’s when it clicked—this isn’t a one-off bug. It’s how the system behaves."

## Timing and Delivery
- keep pacing tight
- one short pause before the example
- one pause after the example for recognition

## Transition from Slide 3

Slide 3 ends with the consequence ("Never reaches the ending").

Transition line:

"I kept running into this across everything I built with AI…"

Then show this slide.

## Transition to Slide 5 (Failure Pattern)

After the example, move directly into pattern explanation:

"And it wasn’t random. It kept breaking in the same ways."

This sets up the failure pattern slide.

## Do Not
- do not list product names on the slide
- do not include logos
- do not add more than one example
- do not over-explain the bug
- do not turn this into a case study

## Success Criteria
The slide is successful if:
- audience believes this is a real, repeatable issue
- audience understands it happens across tools
- the concrete example feels familiar and specific
- credibility is established without slowing momentum
